- dp数组如何初始化

DP（动态规划）的初始化思想是‌确定递推过程的起点和边界条件‌
初始化的本质是为最简单、最基础的子问题赋予确定的值，这些值将作为后续状态转移的“种子”。

核心思想
‌识别基础情况‌：分析问题，找出那些无法再被拆分成更小子问题的“最简状态”。这些状态的值通常是已知的、固定的，或者根据问题定义直接得出。
‌赋予合理初值‌：根据问题的目标（求最大值、最小值或方案数），为这些基础状态赋予合适的初始值。
‌求最值问题‌：若目标是求‌最大值‌，通常将 dp 数组初始化为一个极小的值（如负无穷或0）；若目标是求‌最小值‌，则初始化为一个极大的值（如正无穷）。
‌求方案数问题‌：通常将 dp 或 dp 初始化为 1，表示“什么都不做”也是一种方案。
‌确保状态转移的可行性‌：初始化的值必须能支撑后续的状态转移方程。例如，在计算 dp[i] 时，它可能依赖于 dp[i-1] 或 dp[i-2]，那么 dp 和 dp‌:ml-citation{ref="1" data="citationList"} 就必须被正确初始化。
具体示例
‌斐波那契数列‌：F(n) = F(n-1) + F(n-2)。最基础的情况是 F(0) = 0 和 F(1) = 1。这两个值就是初始化的边界条件。‌

## 最长严格单调递增子序列

最优时间时间复杂度

利用贪心算法+二分法

[最长递增子序列（nlogn 二分法、DAG 模型 和 延伸问题） | 春水煎茶](https://hit9.dev/post/longest-increasing-subsequence-revisited)

```python
def bisnary_search(arr,target):
    # 假设区间为左闭右开 [ l , r )
    l = 0
    r = len(arr)
    # 为保证区间内整数不为空，l<r
    while l < r:
        m = l + (r-l)//2   #虽然pyhton不会int溢出，但是为了规范，要写
        if target <= arr[m]:
            r = m            # 因为右边是开区间，所以可以取m
        elif arr[m] < target:
            l = m+1            # 因为左边是闭区间
    # 循环结束的条件 l=r，此时arr[m] < target < arr[l]
    # 返回第一个大于等于target的位置,使用l、r都可以，但语义为target的值在左侧
    # target <= arr[ r , len )
    return l


def inc_max(list_h):
    # 初始化，最小递增子序列长度为1
    dp = [ 1 for i in range(len(list_h))]
    # 贪心算法数组 ，末尾元素是最长子序列的末尾元素，长度为该序列的长度
    # 初始化，加入序列的第一个元素*
    rec = [list_h[0]]
    for i in range(1,len(list_h)):
        if list_h[i] > rec[-1]:
            rec.append(list_h[i])
            dp[i] = len(rec)
        else:
            pos = bisnary_search(rec, list_h[i])
            dp[i] = pos + 1
            rec[pos] = list_h[i]
    return dp
```

## 最大公共子串

1 使用二维dp矩阵，i，j分别映射字符串的一位
状态转移方程：
s1[i] == s2[j] ; dp[i][j] = dp[i-1][j-1]+1
else: dp[i][j] = 0
意义：如果字符相同，则根据他们前一位的字符的判定情况加1

## 最长公共子序列

1 使用二维dp矩阵，i，j分别映射字符串的前i,j位的子串公共子序列
状态转移方程：
s1[i-1] == s2[j-1] ; dp[i][j] = dp[i-1][j-1]+1
else: dp[i][j] = max(dp[i-1][j], dp[i][j-1])
意义：如果字符相同，则查看他们之前的子串的公共子序列，并加1
如果不相同，则等于取之前计算好的子串的最大值

```python
# ============================================================
# 动态规划求最长公共子序列 (LCS)
# 细节处理：
# 使用 (n+1) x (m+1) 的表格，避免边界判断；应用后一位记录前一位长度。
# dp[i][j] 表示 s[:i] 和 t[:j] 的 LCS 长度
# ============================================================
dp = [[0] * (m + 1) for _ in range(n + 1)]

for i in range(1, n + 1):
    for j in range(1, m + 1):
        if s[i-1] == t[j-1]:
            # 字符相同：左上角的值 + 1
            dp[i][j] = dp[i-1][j-1] + 1
        else:
            # 字符不同：取上方或左方的最大值
            dp[i][j] = max(dp[i-1][j], dp[i][j-1])
```

## 分苹果

问题：把m个相同苹果分给n个盘子，盘子可空。（注意苹果必须分完）

思路：把问题缩减为更小的问题（分i个苹果，j个盘子）

初始化：分0,1个苹果； 分一个盘子

推导：

1 苹果不够分的情况

2 苹果够分的情况： 要不要有空盘子的情况

```python
import sys

m,n = list(map(int,input().split())) ]
# dp[i][j] 代表把i个苹果放入j个盘子中，注意不允许苹果不放的情况
dp = [[0] * (n+1) for i in range(m+1
# 初始化
# 0个或1个苹果放入j个盘子中
for i in range(1,n+1):
    dp[0][i] = 1
    dp[1][i] = 1
# i个苹果放入1个盘子中
for i in range(m+1):
    dp[i][1] = 1

# 注意已经初始化只有0、1个苹果的情况，所以应该从2开始.盘子也是同样的情况
for i in range(2,m+1):
    for j in range(2,n+1):
        # 苹果数小于盘子数，意味着必有空盘子。方案数则与少一个盘子的情况相同
        if i < j:
            dp[i][j] = dp[i][j-1]
        # 苹果数大于盘子数
        elif i>=j:
            # 1 若有空盘子的情况，则与少一个盘子的情况
            # 2 若无空盘子的情况,则意味着每个盘子必有一个苹果,
            # 若每个盘子减少一个苹果,方案数不变
            dp[i][j] = dp[i][j-1] + dp[i-j][j] 

print(dp[m][n])
```

## 最优组合（这个问题可以被建模为一个图论问题，具体来说，是一个在二分图上求最大权值独立集的问题）

### 与DFS的些许差异
这是一个非常好的问题，它触及到了对问题本质理解的关键点。将这个问题看作是“从一个起始点出发，进行某种路径遍历直到无路可走”的想法很自然，但这是不正确的，原因如下：

1.  **问题的目标是“集合”而非“路径”**：
    *   原问题要求我们从矩阵中**取出若干个数字**，形成一个**集合**。这个集合中的数字必须满足“互不相邻”的约束，目标是最大化这个集合中所有数字的总和。
    *   而“路径遍历”的思路，无论是DFS还是BFS，其本质是探索一个**有序的序列**（路径）。例如，从A点走到B点，再到C点... 这个序列是有先后顺序的。

2.  **“路径遍历”会遗漏最优解**：
    *   **最优解可能不是连通的**：想象一个3x3的矩阵，最优的选择可能是角上的四个点 `(0,0), (0,2), (2,0), (2,2)`。这四个点在矩阵上是分离的，不存在一条从一个点出发，只访问这四个点的“路径”。任何从单一固定起点出发的路径搜索都无法找到这个解，因为它在找到一个点后，会强迫你沿着相邻的点移动，从而无法跳到对角线上去。
    *   **“路径遍历”限制了选择的自由度**：路径遍历算法（如DFS）会遵循一个特定的探索顺序（例如，进入一个点后，优先探索它的上、下、左、右）。这种顺序决定了你只能找到那些符合该顺序的解。然而，最优解 `{(0,0), (0,2)}` 和解 `{(0,1), (2,1)}` 之间没有路径上的联系，路径遍历算法无法轻易地在这两种可能性之间进行比较和切换。

3.  **正确的思路是“状态枚举”**：
    *   正确的解法（如前面的代码）是进行**状态空间的搜索**。对于矩阵中的**每一个单元格**，我们都考虑两种可能性：“放入最终集合”和“不放入最终集合”。
    *   这是一个典型的**组合优化**问题。我们并不是在寻找一条路，而是在为矩阵中的每一个位置做“是”或“否”的决策，从而枚举出所有可能的“数字集合”（在约束条件下）。
    *   这种“对每个元素做决策”的思路，通常对应着“选或不选”的递归结构，而不是“从一点走向另一点”的路径结构。

**总结**:

*   **原问题**：从矩阵中选出一个**数字集合**，集合内数字不相邻，求最大和。这是一个**组合问题**。
*   **“路径遍历”思路**：从一个点出发，按照规则（不相邻）移动，形成一个**数字序列**。这是一个**路径问题**。

两者的目标和约束虽然表面上有关联（都是“不相邻”），但问题的结构完全不同。路径遍历的搜索空间远小于组合问题的搜索空间，因此会不可避免地错过最优解。



### 动态规划或状态压缩搜索问题

HJ152 取数游戏
好的，下面是使用 Python 实现的“取数游戏”解决方案，代码中包含了大量注释，方便您理解每一步的逻辑。

```python
import sys

# 为了防止在某些情况下递归过深导致栈溢出，
# 我们可以适当增加递归限制（对于本题给定的数据规模，通常是不必要的）
# sys.setrecursionlimit(2000)

def solve_game():
    """
    主函数，处理单个测试用例的输入、调用DFS求解并返回结果。
    """
    # 读取矩阵的行数 N 和列数 M
    n, m = map(int, input().split())
    
    # 创建一个 N 行 M 列的矩阵，用于存储数字
    matrix = []
    for i in range(n):
        row = list(map(int, input().split()))
        matrix.append(row)

    # 创建一个 N 行 M 列的布尔矩阵，用于记录某个位置的数字是否被选中
    # `is_selected[r][c]` 为 True 表示 (r, c) 位置的数字被选中了
    is_selected = [[False] * m for _ in range(n)]

    # 用于存储在整个DFS搜索过程中找到的最大和
    # 初始化为负无穷大，确保任何有效结果都能更新它
    max_sum_found = float('-inf')

    def is_valid(r, c):
        """辅助函数：检查坐标 (r, c) 是否在矩阵的有效范围内"""
        return 0 <= r < n and 0 <= c < m

    def can_place_at(row, col):
        """
        辅助函数：检查在 (row, col) 位置放置一个数字是否可行。
        即，检查其周围8个方向的已选中数字（如果存在）。
        由于我们是按行优先（从左到右，从上到下）遍历的，
        所以只需检查左、左上、上、右上这四个方向即可。
        """
        # 定义8个方向的偏移量 (行偏移, 列偏移)
        # 这里只检查我们已经遍历过的方向
        neighbors_offsets = [
            (0, -1),   # 左
            (-1, -1),  # 左上
            (-1, 0),   # 上
            (-1, 1)    # 右上
        ]
        
        for dr, dc in neighbors_offsets:
            neighbor_r, neighbor_c = row + dr, col + dc
            
            # 检查邻居坐标是否有效，并且是否已经被选中
            if is_valid(neighbor_r, neighbor_c) and is_selected[neighbor_r][neighbor_c]:
                # 如果任何一个邻居被选中了，则不能在此处放置
                return False
        
        # 所有邻居都没被选中，可以放置
        return True

    def dfs(current_row, current_col, current_sum):
        """
        深度优先搜索函数
        
        :param current_row: 当前正在处理的行索引
        :param current_col: 当前正在处理的列索引
        :param current_sum: 在这条搜索路径上，到目前为止已经累加的数字和
        """
        nonlocal max_sum_found # 声明 max_sum_found 是外层函数的变量，允许在内部修改它

        # --- 步骤1: 确保坐标不越界 ---
        # 如果当前列超过了最后一列，说明这一行已经处理完了
        # 需要换到下一行的第一列继续处理
        if current_col >= m:
            current_row += 1
            current_col = 0

        # --- 步骤2: 检查是否已遍历完所有单元格 ---
        # 如果行索引超出了矩阵的最后一行，说明整个矩阵都已处理完毕
        # 这是一个递归的终止条件，更新全局最大值
        if current_row >= n:
            max_sum_found = max(max_sum_found, current_sum)
            return # 结束当前递归分支

        # --- 步骤3: 对当前位置做出选择 ---
        # 选择一：不选择当前位置 (current_row, current_col) 的数字
        # 直接跳到下一个位置继续搜索
        # 这是最简单的情况，没有约束
        dfs(current_row, current_col + 1, current_sum)

        # 选择二：选择当前位置 (current_row, current_col) 的数字
        # 但前提是选择它是合法的（即与相邻的已选数字不冲突）
        if can_place_at(current_row, current_col):
            # 1. 将当前位置标记为已选中
            is_selected[current_row][current_col] = True
            
            # 2. 将当前位置的数字加到当前路径的总和里
            new_sum = current_sum + matrix[current_row][current_col]
            
            # 3. 递归处理下一个位置
            dfs(current_row, current_col + 1, new_sum)

            # 4. 关键的“回溯”步骤
            # 递归返回后，必须撤销刚才的选择，恢复现场
            # 这样才能让后续的递归分支能够自由地选择这个位置
            is_selected[current_row][current_col] = False

    # --- 启动DFS搜索 ---
    # 从矩阵的第一个位置 (0, 0) 开始，初始总和为 0
    dfs(0, 0, 0)

    # 返回在整个搜索过程中找到的最大和
    return max_sum_found


def main():
    """
    程序入口函数，处理多组测试用例。
    """
    # 读取总的测试用例数量 T
    t = int(input())
    
    # 循环处理每一个测试用例
    for _ in range(t):
        result = solve_game()
        print(result)

# 当此脚本作为主程序运行时，执行 main() 函数
if __name__ == "__main__":
    main()
```

**代码逻辑详解:**

1.  **`solve_game()`**: 这是处理单个游戏实例的函数。它负责读取矩阵数据，并初始化用于搜索的变量。
2.  **`is_selected` 矩阵**: 这是解决问题的关键。它就像一张“地图”，记录了在当前的搜索路径中，哪些数字已经被选中了。这样在决定是否选择一个新的数字时，就能快速检查其邻居。
3.  **`can_place_at(row, col)`**: 这个辅助函数实现了题目的核心约束。它只检查 `(row, col)` 位置的“已访问”邻居（左、左上、上、右上），因为我们是按顺序遍历的，其他方向的邻居还没处理到。
4.  **`dfs(current_row, current_col, current_sum)`**: 这是核心的递归函数。
    *   **`nonlocal max_sum_found`**: 这句代码非常重要，它告诉Python `max_sum_found` 不是`dfs`函数内部的局部变量，而是它外面一层函数（`solve_game`）里的变量，这样`dfs`函数内部就可以修改它了。
    *   **递归终止**: 当行索引超出矩阵范围时，说明我们已经考察了所有单元格，当前的 `current_sum` 就是一个有效的总和，我们用它来更新全局的 `max_sum_found`。
    *   **两个选择**: 函数明确地模拟了两种选择：不选 `dfs(current_row, current_col + 1, current_sum)` 和选 `dfs(current_row, current_col + 1, new_sum)`。
    *   **回溯 (`is_selected[current_row][current_col] = False`)**: 这是回溯算法的灵魂。在递归调用返回后，必须撤销之前的修改（即将标记设回 `False`），这样才能保证搜索树的其他分支能够不受当前路径的影响，从而完整地探索所有可能性。
