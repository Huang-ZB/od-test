# 无向图

## 连通性
连通图： 无向图中，任意两个节点可以相互到达
连通分量：每个极大流通子图可以作为一个连通分量

## 性质
### 无向图含有n个顶点，图的边数
n-1(树) <=  边数  <=  (n-1)(n-2)//2 +1 (n-1个顶点的完全无向图 连接1个顶点组成的图)
### 总路径数
（树中任意两点间只有一条路径，即不存在环且两个节点之间最多一条边）
total_paths = n * (n - 1) // 2

# 二分图

描述：

二分图：如果一张图可以被染为两种不同的颜色，并且一条边两端的节点颜色不同

匹配：

在一个图中，一个节点可以存在多条边（代表多种匹配的方案）

匹配边：表示两点联通

非匹配边：表示存在该联通的方案，但此时并未匹配

例子：乘客和座位可以理解为不同的节点，为一个乘客分配一个座位称为一个匹配

交替路：一条路径上非匹配边和匹配边交替出现

增广路：查找一条交替路，若反转其匹配关系，可使其匹配数加1，（实际是以非匹配边开始，也以非匹配边结束的交替路）

## 二分图的最大匹配

### 思路
遍历一组的节点，对每个节点使用dfs查找增广路。

# 极大团

## 使用 Bron–Kerbosch 算法（带 pivot 优化）找出所有极大团

### 模版
```python
def bron_kerbosch_with_pivot(graph):
    """
    参数:
        graph: dict，邻接表表示的无向图。
               例如: {'A': {'B', 'C'}, 'B': {'A', 'C'}, 'C': {'A', 'B'}}
               每个节点的邻居用 set 存储，便于集合运算。
    
    返回:
        cliques: list of sets，每个 set 是一个极大团。
    """
    cliques = []  # 用于存储所有找到的极大团

    def bk(r, p, x):
        """
        递归核心函数。
        
        参数含义（经典 Bron–Kerbosch 三集合）：
          - R (r): 当前正在构建的团（已选节点）
          - P (p): 可能加入 R 的候选节点（与 R 中所有节点都相邻）
          - X (x): 已排除的节点（与 R 中所有节点都相邻，但已处理过，避免重复）
        
        不变量：R ∪ P ∪ X 中的任意节点都与 R 中所有节点相邻。(∪并集)
        目标：当 P 和 X 都为空时，R 就是一个极大团。
        """
        # 终止条件：如果 P 和 X 都为空，说明 R 无法再扩展，且没有遗漏 → R 是极大团
        if not p and not x:
            cliques.append(r.copy())  # 保存当前极大团（注意要 copy！）
            return

        # 选择一个 pivot 节点 u，从 P ∪ X 中任选（通常选度数高即邻接点多的的可更好剪枝）
        # 目的：减少后续遍历的候选节点数量
        if p or x:
            # 从 P ∪ X 中任取一个元素作为 pivot（这里用 next(iter(...)) 快速取一个）
            u = next(iter(p | x))
            # 优化关键：只遍历 P 中 **不是 u 的邻居** 的节点
            # 因为对于 u 的邻居 v，其可能产生的团会被其他分支覆盖（避免冗余）
            candidates = p - graph[u]
        else:
            # 理论上不会走到这里，因为上面 if 已判断 p or x
            candidates = p

        # 🔁 遍历所有候选节点 v
        for v in list(candidates):  # 转为 list 避免在迭代时修改 set
            # 递归：将 v 加入当前团 R
            # 新的候选集 = P ∩ N(v) （只保留与 v 相邻的节点）
            # 新的排除集 = X ∩ N(v)
            bk(
                r | {v},              # R ∪ {v}
                p & graph[v],         # P ∩ neighbors(v)
                x & graph[v]          # X ∩ neighbors(v)
            )
            # 回溯：将 v 从 P 移到 X（表示 v 已处理完毕，不再考虑）
            p.remove(v)
            x.add(v)

    # 初始调用：R=∅, P=所有节点, X=∅
    all_nodes = set(graph.keys())
    bk(set(), all_nodes, set())

    return cliques
```

### 理解
https://blog.csdn.net/weixin_28338005/article/details/158684001
1. 对X集合的作用进行模拟
   假设abc，abd为一个图的两个极大团.设N(v)为v的邻接点集合
   对于子团bc，(abc)∩N(b) = (ac) , (ac)∩N(c) = (a);  或者 不严谨的有 N(bc) = (a),(abc)∩N(bc) = (a)
   结论：子团的内部元素的公共邻接店必然存在，与极大团的交集必然非空，即X非空，实现排除的效果
   推导：设两个极大团x，y，它们有公共的子团z，分别x，y与N(z)进行交集，剩余除z以外的元素(相当于递归栈中的若干步)
   ，后续再进行交集，X必然为空。(abc)回溯到ab状态时X：(c)，在ab状态下再递归d的分支，N(d)∩(c) = ∅
2. pivot优化
   每层递归，会遍历对应的P集合内的元素。
   在朴素的BK算法中，假设第i层深度的递归树中选择了u节点，对于包含u的极大团，N(u)也会有一部分在该团中，所以后续的递归会包含了N(u)的分支。即可能出现选择路径[u,N(u)]
   若第i层深度的递归树中遍历到v节点，而v属于N(u)的集合中，那可能会出现以下选择路径[v,u,N(u)]。 [u,N(u)]中也存在该路径的选择，即是一个重复的子问题。
   所以某层遍历，当选择u节点时，后续的路径必定也会搜索到N(u)节点，而不会遗漏。所以该层可以通过剪枝只留下u即可。
